# Fitness Assistant - API Documentation

Все запросы работают на http://localhost:8080

## 1. Поиск упражнений по группе мышц

Метод: GET  
URL: /api/exercises  
Параметры: ?muscleGroup=название_мышцы  
Коды ответов: 200, 400, 404  

Пример запроса: GET /api/exercises?muscleGroup=грудь  

Ответ (200):  
{  
  "muscleGroup": "грудь",  
  "exercises": ["Жим лежа", "Разводки гантелей"]  
}

## 2. Поиск группы мышц по упражнению

Метод: GET  
URL: /api/exercises/muscles  
Параметры: ?exercise=название_упражнения  
Коды ответов: 200, 400, 404  

Пример запроса: GET /api/exercises/muscles?exercise=приседания  

Ответ (200):  
{  
  "exercise": "приседания",  
  "muscles": ["квадрицепсы", "ягодицы"]  
}

## 3. Расчет максимального веса

Метод: POST  
URL: /api/calculations/max-weight  
Тело запроса (отправляем JSON):  
{  
  "exercise": "жим лежа",  
  "weight": 80,  
  "reps": 5,  
  "age": 16,  
  "height": 175,  
  "weightPerson": 70  
}  
Коды ответов: 200, 400  

Ответ (200):  
{  
  "exercise": "жим лежа",  
  "maxWeight": 92.5  
}

## 4. Расчет КБЖУ продукта

Метод: POST  
URL: /api/nutrition/calculate  
Тело запроса (отправляем JSON):  
{  
  "foodName": "куриная грудка",  
  "weightGrams": 200  
}  
Коды ответов: 200, 400, 404  

Ответ (200):  
{  
  "food": "куриная грудка",  
  "weight": 200,  
  "calories": 220,  
  "protein": 46,  
  "fat": 5,  
  "carbs": 0  
}

## 5. Просмотр тренировки на сегодня

Метод: GET  
URL: /api/schedule/today  
Коды ответов: 200, 404  

Ответ (200):  
{  
  "day": "Понедельник",  
  "workout": "Грудь и трицепс",  
  "exercises": ["Жим лежа", "Отжимания"]  
}

## 6. Просмотр расписания на неделю

Метод: GET  
URL: /api/schedule/week  
Коды ответов: 200, 404  

Ответ (200):  
{  
  "monday": "Грудь и трицепс",  
  "tuesday": "Спина и бицепс",  
  "wednesday": "Ноги"  
}

## 7. Редактирование расписания

Метод: PUT  
URL: /api/schedule  
Тело запроса (отправляем JSON):  
{  
  "monday": "Грудь",  
  "tuesday": "Спина",  
  "wednesday": "Ноги"  
}  
Коды ответов: 200, 400  

Ответ (200):  
{  
  "message": "Расписание обновлено"  
}

## 8. Проверка работы сервера

Метод: GET  
URL: /api/health  
Коды ответов: 200, 500  

Ответ (200):  
{  
  "status": "OK"  
}
